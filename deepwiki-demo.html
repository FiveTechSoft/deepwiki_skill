This page introduces the FiveWin for Harbour (FWH) repository: its purpose, platform targets, directory layout, and the eight major subsystems. For installation steps see Installation. For detailed class internals see Core Architecture. For build scripts and compiler targets see Build and Deployment.

What is FWH?

FWH is a Windows desktop GUI application framework built on top of the Harbour and xHarbour compilers. It wraps the Win32 API in a class-based object model, letting application code written entirely in Harbour (a dBASE/Clipper-compatible language) create windows, dialogs, controls, menus, database grids, printed reports, and — in recent releases — embedded web browsers and AI service integrations.

The framework version is defined in include/fivewin.ch#L8-9:

#define FWVERSION    "FWH 25.12"
#define FW_VersionNo 25120

The FWDESCRIPTION macro resolves to one of four strings depending on compiler and bitness (include/fivewin.ch#L12-34).

Compiler Bitness Description
Harbour 32-bit "FiveWin for Harbour"
Harbour 64-bit "FiveWin 64 for Harbour"
xHarbour 32-bit "FiveWin for xHarbour"
xHarbour 64-bit "FiveWin 64 for xHarbour"

The single application-level include file is include/fivewin.ch. It aggregates all subsystem headers.

Repository Layout

The project follows a traditional C/Harbour library layout:

  • source/ — Core .prg and .c source files organized by subsystem
  • include/ — Header files (.ch and .h)
  • samples/ — Example applications demonstrating features
  • lib/ — Pre-built libraries for different compiler targets

Data Access Architecture

FWH provides a unified data access layer that supports multiple database backends including DBF/CDX, MySQL/MariaDB, SQLite, PostgreSQL, and ODBC connections. The architecture uses an abstraction layer where TDatabase and TRecordSet classes provide a consistent API regardless of the underlying storage engine.

flowchart LR A["Application Code"] --> B["TDatabase / TRecordSet"] B --> C["DBF/CDX Driver"] B --> D["MySQL/MariaDB Driver"] B --> E["SQLite Driver"] B --> F["ODBC Driver"] C --> G[("Local Files")] D --> H[("MySQL Server")] E --> I[("SQLite File")] F --> J[("Any ODBC Source")]

Build and Platform Support

FWH supports multiple C/C++ compilers and can target both 32-bit and 64-bit Windows platforms. The build system uses makefiles and batch scripts tailored for each compiler.

Getting Started

This section covers how to install FWH, build your first application, and explore the sample programs.

Installation

Download the latest FWH package from the official website. Extract the archive to a directory of your choice (e.g., C:\FWH). Set the FWDIR environment variable: 

SET FWDIR=C:\FWH
SET PATH=%FWDIR%\bin;%PATH%

Hello World Tutorial

Create a file called hello.prg:

#include "FiveWin.ch"

FUNCTION Main()
    MsgInfo("Hello, World!", "FWH")
RETURN NIL

Build and run:

buildh hello
hello.exe

Core Architecture

The core class hierarchy forms the backbone of every FWH application. All visual elements inherit from TWindow, which wraps a Windows HWND handle.

Core Class Hierarchy

Window System (TWindow)

TWindow is the foundational class for all FWH applications. It manages the Windows message loop, painting, resizing, and event dispatching. Every visible element — dialogs, controls, MDI frames — ultimately inherits from TWindow.

Event Handling

FWH uses a code-block delegation model for event handling. Controls expose bAction, bWhen, bValid, and other code-block data members that application code sets to customize behavior:

@ 10, 10 BUTTON oBtn PROMPT "Click Me" SIZE 100, 30 ;
   ACTION MsgInfo("Button clicked!")

AI and Modern Features

📄 Relevant source files ›
📄 source/classes/ai.prg 📄 source/classes/webview.prg

Recent FWH releases integrate AI services (OpenAI, Google Gemini, Anthropic Claude) and modern web technologies via embedded Chromium/Edge browsers.

AI Integration

The TAI class provides a simple interface to interact with LLM APIs directly from Harbour code:

oAI := TAI():New("openai", cApiKey)
cResponse := oAI:Chat("Explain FiveWin in one paragraph")

WebView Integration

The TWebView class embeds a modern Chromium-based browser control, enabling hybrid desktop/web applications with bidirectional communication between Harbour and JavaScript.
